# Guia de Contribuição

Obrigado por considerar contribuir com a tradução PT-BR de Oxygen Not Included!
Este documento explica como você pode ajudar a melhorar o projeto caso ja tenha algum conhecimento.

## Tipos de contribuição

Você pode contribuir de várias formas:

- 📝 Correção de traduções.
- 🌐 Tradução de novos textos.
- 🔍 Revisão de textos existentes.
- 🐞 Reportar erros (issues).
- 💡 Sugestões de melhoria.

## Começando

Faça um fork do repositório e clone para sua máquina:

```commandline
 git clone https://github.com/SEU-USUARIO/oni-traducao-pt-br.git
```

Entre na pasta:

```commandline
cd oni-traducao-pt-br
```

### Criando uma branch:

Sempre crie uma branch para suas alterações:

```commandline
git checkout -b feat/minha-contribuicao
```

Exemplos:

- fix/termo-oxigenio - Correções de termos ou erros de gramatica.
- feat/novas-strings - Novas melhorias ou adição de contéudo. 
- review/padronizacao - Revisões.

## Padrões de tradução

Para manter qualidade e consistência:

✔️ Faça:
- Use português natural (PT-BR).
- Priorize contexto ao invés de tradução literal.
- Mantenha consistência de termos.
- Revise ortografia e concordância.

❌ Evite:
- Traduções literais sem sentido.
- Misturar PT-BR com PT-PT ou termos EN-US com EN-UK.
- Alterar termos técnicos já consolidados ou nomeclatura de estruturas, plantas ou animais.

## Commits

Use mensagens claras e objetivas:

git commit -m "fix: corrige traducao de 'Oxygen Generator'"

Padrão sugerido:

- fix: correções.
- feat: novas traduções.
- review: revisões.
- refactor: melhorias estruturais.
## Enviando alterações

```commandline
git push origin minha-contribuicao
```

Depois:

Abra um Pull Request.
Descreva claramente o que foi feito, porfim aguarde revisão.

## Reportando problemas

Se encontrar erros:

1- Vá até a aba Issues.

2- clique em 'New Issue' inclua: 
        
- Descrição do problema, 
- Texto original preferidamente prints, 
- Tradução atual preferidamente prints, 
- Sugestão (se possível).

## Revisão
Todas as contribuições passam por revisão antes de serem aceitas.
Podemos sugerir ajustes para manter o padrão do projeto.

## Boas práticas

- Faça alterações pequenas e focadas.
- Evite commits gigantes e/ou com grande impacto.
- Teste no jogo sempre que possível. 🎮
- Mais Importante: Seja respeitoso nas discussões todos estamos aqui para melhorar um jogo que gostamos.

## ❤️ Agradecimentos

Cada contribuição ajuda a melhorar a experiência de toda a comunidade brasileira de oni.
Muito obrigado por participar! 🇧🇷🚀
